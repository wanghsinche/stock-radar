# SP500 20日新高 + 120日动量轮动策略

## 策略逻辑

```
每周五收盘后生成信号，周一开盘执行:
1. 扫描 SP500，选过去20日内创20日新高的股票
2. 按 (当前价 / 120日前价 - 1) 降序排名，取前20为候选池
3. 买入池中前10只
4. 持有池中第11-20只（不新买，不卖）
5. 卖出已跌出前20的持仓
6. 回测成交价按下个交易日开盘价 + 0.1% 滑点
```

### 仓位规则

- 初始资金 $20,000，买入目标为 top10。
- 每年首次运行时更新新买入预算：`position_budget = int(组合总资产 / 10)`。
- 年内沿用该预算，只影响新买入，不做年度强制清仓重配。
- 因为 top10 买入、mid10 继续持有，实际持仓数可能超过 10；历史回测最大约 19 只。

### 安全机制

| 机制 | 条件 | 动作 |
|------|------|------|
| **空仓** | 合格数 < 15 | 不买，现金持有 |
| **认输转 SPY** | 策略从峰值回撤 > 15% | 清仓全仓 SPY |
| **转回股票** | SPY > 50日均线 且 合格数 >= 30 | 卖 SPY 重新买股 |
| **冲突换仓** | 同时满足认输和转回条件 | 跳过 SPY，直接强制换到当周 top10 |
| **冷却期** | 转回股票后 4 周内 | 不再触发认输 |

## 回测结果

```
数据范围: 2021-06-28 → 2026-07-20
回测运行区间: 2021-07-09 → 2026-07-10 (253周)
初始资金: $20,000
SP500 成分股: 503只
执行假设: 周一开盘价成交 + 0.1%滑点
```

| 区间 | 策略 | SPY 买入持有 | 最大回撤 |
|------|------|-------------|---------|
| 全周期 5年 | **+379.78%** | +86.24% | **-27.01%** |
| 牛市 2025-04→09 | **+26.90%** | +25.02% | -8.01% |
| 熊市 2022-01→06 | **+7.97%** | -17.67% | -10.83% |
| 猴市 2025-10→2026-04 | **+77.01%** | +6.72% | -10.88% |
| 最大回撤期 2021-11→2023-03 | **-4.72%** | -12.02% | -22.09% |

### 关键指标（全周期）

- 终值: $95,956
- 年化收益: 36.82%
- 最大回撤: -27.01%
- 胜率: 54.15%
- 平均持股: 2.5 周
- 交易次数: 870 买 / 860 卖（5年）
- 年度新买入预算期末约 $5,500/只

### 关键实验记录

- 固定 $2,000/只：+226.83%，期末资金利用率约 32.2%。
- 每年更新新买入预算为组合总资产/10：+379.78%。
- 每年强制全量重配 top10：+361.42%，不如只更新新买入预算。
- 防守标的换 SCHD（信号仍用 SPY）：+198.82%，不如 SPY 防守。
- 冲突处理：跳过 SPY 并强制换仓 +226.83%；直接 skip 不交易 +219.21%；同日买卖 SPY +224.75%。

## 运行方式

```bash
cd stock-radar

# 运行每周扫描
uv run python src/main.py

# 运行完整回测
uv run python src/backtest.py

# 仅扫描（不推送到 Telegram）
uv run python -c "from src.scanner import scan_and_save; scan_and_save()"
```

### 定时任务（pm2）

每周六 06:00（北京时间）自动运行：

```bash
pm2 start ecosystem.config.js
```

### 配置

- `config.yaml`: `radar.top_n` 控制候选池大小（默认 20）
- `.env`: 存储 Telegram bot token 和 chat_id

## 文件结构

```
stock-radar/
├── src/
│   ├── scanner.py       # SP500 成分股获取 + 20日新高筛选
│   ├── backtest.py      # 周频轮动回测 + 权益曲线
│   ├── notifier.py      # Telegram 消息 + 图片推送
│   └── main.py          # 扫描 + 推送入口
├── config.yaml          # 策略配置
├── ecosystem.config.js  # pm2 cron
├── reports/backtest/    # 回测图表输出
└── .env                 # Telegram 凭证（gitignored）
```

## 参数调优

核心参数在 `src/backtest.py` 的 `run_backtest()`:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `top_n` | 20 | 候选池大小 |
| `buy_top` | 10 | 每周买入目标数量，也是年度预算分母 |
| `min_qualify_half` | 15 | 合格数低于此值不买 |
| `position_budget` | 总资产 / 10 | 每年首次运行更新，只影响新买入 |
| `dd_switch_to_spy` | 0.15 | 回撤超过此比例切换到 SPY |
| `reentry_min_qual` | 30 | 转回股票所需最低合格数 |
| `reentry_spy_ma` | 50 | 转回股票时 SPY 需站上此均线 |
| `spy_cooldown_weeks` | 4 | 转回股票后冷却周数 |
