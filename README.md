# SP500 20日新高 + 120日动量轮动策略

## 策略逻辑

```
每周五收盘后生成信号，周一开盘执行:
1. 扫描 SP500，选过去20日内创20日新高的股票
2. 按 (当前价 / 120日前价 - 1) 降序排名，取前20为候选池
3. 买入池中前10只
4. 持有池中第11-20只（不新买，不卖）
5. 卖出已跌出前20的持仓
6. 回测成交价按下个交易日开盘价 + 0.1% 滑点
```

### 仓位规则

- 初始资金 $20,000，买入目标为 top10。
- 每年首次运行时更新新买入预算：`position_budget = int(组合总资产 / 10)`。
- 年内沿用该预算，只影响新买入，不做年度强制清仓重配。
- 因为 top10 买入、mid10 继续持有，实际持仓数可能超过 10；5年回测最大 16 只，但总仓位仍受现金约束，不使用杠杆。

### 安全机制

| 机制 | 条件 | 动作 |
|------|------|------|
| **空仓** | 合格数 < 15 | 不买，现金持有 |
| **认输转 SPY** | 策略从峰值回撤 > 15% | 清仓全仓 SPY |
| **转回股票** | SPY > 50日均线 且 合格数 >= 30 | 卖 SPY 重新买股 |
| **冲突换仓** | 同时满足认输和转回条件 | 跳过 SPY，直接强制换到当周 top10 |
| **冷却期** | 转回股票后 4 周内 | 不再触发认输 |

## 运行方式

```bash
cd stock-radar

# 运行每周扫描
uv run python src/main.py

# 运行完整回测
uv run python src/backtest.py

# 仅扫描（不推送到 Telegram）
uv run python -c "from src.scanner import scan_and_save; scan_and_save()"
```

### 定时任务（pm2）

每周六 06:00（北京时间）自动运行：

```bash
pm2 start ecosystem.config.js
```

### 配置

- `config.yaml`: 运行环境配置
- `.env`: 存储 Telegram bot token 和 chat_id

## 文件结构

```
stock-radar/
├── src/
│   ├── scanner.py       # SP500 成分股获取 + 20日新高筛选
│   ├── backtest.py      # 周频轮动回测 + 权益曲线
│   ├── notifier.py      # Telegram 消息 + 图片推送
│   └── main.py          # 扫描 + 推送入口
├── config.yaml          # 策略配置
├── ecosystem.config.js  # pm2 cron
├── reports/backtest/    # 回测图表输出
└── .env                 # Telegram 凭证（gitignored）
```

## 参数调优

核心参数在 `src/backtest.py` 的 `run_backtest()`:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `top_n` | 20 | 候选池大小 |
| `buy_top` | 10 | 每周买入目标数量，也是年度预算分母 |
| `min_qualify_half` | 15 | 合格数低于此值不买 |
| `position_budget` | 总资产 / 10 | 每年首次运行更新，只影响新买入 |
| `dd_switch_to_spy` | 0.15 | 回撤超过此比例切换到 SPY |
| `reentry_min_qual` | 30 | 转回股票所需最低合格数 |
| `reentry_spy_ma` | 50 | 转回股票时 SPY 需站上此均线 |
| `spy_cooldown_weeks` | 4 | 转回股票后冷却周数 |
