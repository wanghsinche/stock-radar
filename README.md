# SP500 20日新高轮动策略

## 策略逻辑

```
每周五收盘后:
1. 扫描 SP500，选过去20日内创20日新高的股票
2. 按 (当前价 / 20日前价 - 1) 降序排名，取前20为候选池
3. 买入池中前10只（每只 $2000）
4. 持有池中第11-20只（不新买，不卖）
5. 卖出已跌出前20的持仓
```

### 安全机制

| 机制 | 条件 | 动作 |
|------|------|------|
| **减仓** | 合格数 < 15 | 只买 5 只 |
| **空仓** | 合格数 < 5 | 不买，现金持有 |
| **认输转 SPY** | 策略从峰值回撤 > 15% | 清仓全仓 SPY |
| **转回股票** | SPY > 60日均线 且 合格数 >= 30 | 卖 SPY 重新买股 |
| **冷却期** | 转回股票后 4 周内 | 不再触发认输 |

## 运行方式

```bash
cd stock-radar

# 运行每周扫描
uv run python src/main.py

# 运行完整回测
uv run python src/backtest.py

# 仅扫描（不推送到 Telegram）
uv run python -c "from src.scanner import scan_and_save; scan_and_save()"
```

### 定时任务（pm2）

每周六 06:00（北京时间）自动运行：

```bash
pm2 start ecosystem.config.js
```

### 配置

- `config.yaml`: `radar.top_n` 控制候选池大小（默认 20）
- `.env`: 存储 Telegram bot token 和 chat_id

## 文件结构

```
stock-radar/
├── src/
│   ├── scanner.py       # SP500 成分股获取 + 20日新高筛选
│   ├── backtest.py      # 周频轮动回测 + 权益曲线
│   ├── notifier.py      # Telegram 消息 + 图片推送
│   └── main.py          # 扫描 + 推送入口
├── config.yaml          # 策略配置
├── ecosystem.config.js  # pm2 cron
├── reports/backtest/    # 回测图表输出
└── .env                 # Telegram 凭证（gitignored）
```

## 参数调优

核心参数在 `src/backtest.py` 的 `run_backtest()`:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `top_n` | 20 | 候选池大小 |
| `buy_top` | 10 | 每周买入数量 |
| `min_qualify_half` | 15 | 合格数低于此值只买一半 |
| `min_qualify_cash` | 5 | 合格数低于此值空仓 |
| `dd_switch_to_spy` | 0.15 | 回撤超过此比例切换到 SPY |
| `reentry_min_qual` | 30 | 转回股票所需最低合格数 |
| `reentry_spy_ma` | 50 | 转回股票时 SPY 需站上此均线 |
| `spy_cooldown_weeks` | 4 | 转回股票后冷却周数 |
